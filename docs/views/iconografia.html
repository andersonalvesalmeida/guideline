O ícone é uma representação que deve estar em consonância com a cultura da empresa, ou seja, ele deve ser um facilitador de entendimento. O ícone deve ser colocado junto a um botão para que em algum momento do sistema ele possa aparecer sozinho (com texto em mouse over) e continuar fazendo sentido. O ícone deve estar sempre à esquerda do texto que o acompanha (no caso de botões). Um exemplo é o o projeto SIGIE, em que a unidade deverá se guiar apenaspelos ícones 'editar' e 'excluir' ao fazer o planejamento.

Utilizando um sprite

Em vez de fazer todos os ícones separados, nós compilamos eles em um único sprite—um monte de imagens em um arquivo que usa o CSS para posicionar as imagens com background-position. Este é o mesmo método que usamos no Twitter.com.

Todas as classes dos ícones são prefixadas com .glyphicon glyphicon- para o espaçamento entre nomes próprios, bem como em outros componentes. Isso ajudará a evitar conflitos com outras ferramentas.

Como usar

A Guideline usa uma tag<span> para todos os ícones, mas eles não têm nenhum caso de classe, apenas um prefixo comum. Para usar, coloque o seguinte código em qualquer lugar:

<span class="glyphicon glyphicon-search"></span>

Existem também estilos disponíveis para ícones invertidos (brancos), que podem ser utilizados a partir de uma classe extra:

<span class="glyphicon glyphicon-search glyphicon glyphicon-white"></span>

Há 140 classes para escolher a partir de seus ícones. Basta adicionar uma tag <span> com a classe certa e estará pronto. Você pode encontrar o ícone desejado na lista completa em sprites.less ou aqui mesmo no presente documento. Esse sprite de ícones poderá ser aumentado, com isso aumentará o número de classes que representa os ícones.

Mas atenção! Ao utilizar ao lado de áreas de texto, como em botões ou links, não se esqueça de deixar um espaço após a tag <span> para o espaçamento adequado.

Casos de uso

Os ícones são legais, mas onde então devemos usá-los? Aqui estão algumas idéias:

  • Como recursos visuais para a sua barra lateral de navegação
  • Para uma navegação puramente glyphicon glyphicon-driven "navegação orientada pelos ícones"
  • Em botões para ajudar a transmitir o significado de uma ação
  • Com links para compartilhar contexto do destino de um usuário

Essencialmente, qualquer lugar que você pode colocar uma tag <span>, você pode colocar um ícone.
